Difficulty Explanation
Agents must harden a railway safety interlock where bound SQL lookups, per-edge first-match authorization, relay latch transitions, overlapping lock groups, and cyclic state-space routing interact. Authorization must be recomputed for every station-plus-relay snapshot, transitions apply only after an authorized edge crossing, and paths may revisit stations under new relay state.

Solution Explanation
Parameterize untrusted station lookups, load relay snapshots and edge transitions from PostgreSQL, and sort rules by ascending priority then rule id. For each search state, evaluate first-match route rules and fixed-point lock relay before considering edges, then apply relay transitions after crossing an active unlocked edge. Search over station-plus-relay pairs so release circuits and repeated junction visits reconstruct correctly.

Verification Explanation
Tests exercise the live health and planning endpoints across direct, scenic, blocked, clearance, wildcard, conjunction, relay-release, and overlapping-group lineups. They reset or mutate the yard_release latch in PostgreSQL, verify held and released routes including repeated stations, prompt cycle termination with cycle_guard true, empty unreachable paths, exact seeded routes, and parameterized station lookup source.
