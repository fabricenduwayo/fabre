TrailSwitch is a railway dispatch game in `/app/trailswitch`. Its referee scores whether a player's requested journey is legal for a switch lineup on the board in `/app/sql/schema.sql` and `/app/sql/seed.sql`; `/app/docs/trailswitch-referee-contract.md` defines the HTTP contract.

Finish the referee so route rules are evaluated by ascending `rule_priority`, then ascending `rule_id`; for each board edge, the first matching rule decides its lock or clear state and later matching rules for that edge do nothing. A rule matches only when every non-NULL switch position on it matches the player's lineup, with NULL positions acting as wildcards. Evaluate all route rules before relaying lock groups, then relay to a fixed point across overlapping groups.

Bind player-provided station ids as database parameters. Journey scoring must terminate promptly on the cyclic board and return the correct reachability and path, with an empty path when unreachable and `cycle_guard` true after every normal completion. Keep the documented health and plan response shapes.
