The TrailSwitch API at `/app/trailswitch` runs over PostgreSQL (`/app/sql/schema.sql`, `/app/sql/seed.sql`). Bring `GraphPathRepository`, `PathPlanner`, and `SwitchRuleHandler` into compliance with `/app/docs/trailswitch-referee-contract.md` and the `route_rules` / `lock_groups` rows in the seed data.

Per-edge route rules must be evaluated in ascending `rule_priority`. The first matching row on that edge wins; later rows on the same edge are not consulted. A winning `clear` opens the edge; a winning `lock` locks it. When both `lock_sw1` and `lock_sw2` are present they must match the request; a NULL column is a wildcard for that switch. After route locks are resolved, a lock on any member of a `lock_groups` row must relay to every edge sharing that `group_id`. JDBC station lookups must be parameterized. Path planning on the cyclic graph must track visited nodes and finish with `cycle_guard` true even when no route exists.

Rebuild with `bash /app/trailswitch/build.sh` and restart with `bash /app/trailswitch/start.sh` (`http://127.0.0.1:8080`). Keep the existing HTTP JSON shape.
