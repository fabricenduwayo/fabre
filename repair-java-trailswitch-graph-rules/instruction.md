TrailSwitch in `/app/trailswitch` is a railway safety interlock backed by the PostgreSQL board in `/app/sql/schema.sql` and `/app/sql/seed.sql`. Untrusted station ids currently reach SQL unsafely, and incorrect lock precedence can authorize blocked journeys; `/app/docs/trailswitch-referee-contract.md` defines the HTTP contract.

Harden the route authorization so rules are evaluated by ascending `rule_priority`, then ascending `rule_id`; for each edge, the first matching rule decides its lock or clear state and later matching rules for that edge do nothing. A rule matches only when every non-NULL switch position on it matches the request, with NULL positions acting as wildcards. Evaluate all route rules before relaying lock groups, then relay to a fixed point across overlapping groups.

Bind request-provided station ids as database parameters. Route authorization must terminate promptly on the cyclic graph and return the correct reachability and path, with an empty path when unreachable and `cycle_guard` true after every normal completion. Keep the documented health and plan response shapes.
