This task is hard because the rules you need are buried in a long adversarial forensic report with superseded appendices and post-review errata, not in a summary table. The agent has to repair and extend a Java pipeline that slices only operative appendix blocks, walks SQLite audit history with rotation/rescission/replacement semantics, pulls ciphertext out of a GIF, and runs AES-256-GCM with frame_id as AAD. Get the key version or nonce wrong on one frame and authentication fails even if the rest of the code is fine.

I extended the provided three-stage Java CLI under com.mariner.forensic.Main. The rules stage parses operative Appendix C and D for precedence lists and nonce overrides while filtering errata. correlate applies voiding, ordering, and scoping semantics from the report to SQLite audit events. decrypt extracts ciphertext from MRNR/CRYPTO1 GIF blocks and verifies AES-256-GCM using keys from the key_material table. The oracle compiles to /app/pipeline/classes and runs each milestone stage in order.

Each milestone has pytest that reruns the compiled Main subcommand and compares rules.json, correlation.json, and findings.json against hidden fixtures: schema validation, per-frame key/nonce resolution scenarios, withdrawn-errata exclusion, GIF supersession handling, and AES-GCM auth outcomes across all in-scope frames.
