The worker cannot decide anything about an artifact until it has derived which evidence row is operative, and that derivation is a fixpoint rather than a lookup. artifact_evidence is an amendment log, so the agent has to settle amendment authority first, then compute the void set along the supersession chain, then select. Each of those steps changes the answer to the next one. Revocation is not a column any more; it is a replay of key_lifecycle_events at a specific instant, where a key_compromise revoke reaches back to its effective_from and every other reason does not. Trust is then assessed at the operative row's signed_at rather than at run time, so a signature made before a compromise survives and one made after it does not. Finally channel exposure runs as a pass over completed verdicts, which means an artifact signed by a key that was never compromised can still be downgraded because a different artifact in its channel was. Eight mechanics have to compose in the right order, and a shipped test compares all seventeen verdicts at once.

Rewrite Worker.run as three phases instead of one loop. Phase one resolves the operative row for every artifact in the store, not only the queued ones, because channel exposure is scoped from operative signers everywhere. Discard amendments whose amendment_key_id was not live at the amendment's own recorded_at, then void along the chain skipping discarded rows, then take the greatest recorded_at among attested survivors. Phase two evaluates each queued artifact: replay the key events at signed_at for revocation, then check the validity window and the countersignature, then call GET for the detached signature and POST /verify with the operative digest from H2. Phase three walks the finished verdicts and downgrades trusted to quarantine for artifacts in an exposed channel, unless the countersignature covers a signing instant strictly earlier than the exposure. MERGE one row per artifact and commit once.

Pytest drives the live API and the real H2 store, and recomputes the expected manifest at test time from the same inputs the worker reads, so nothing is pinned to a hardcoded winner. Individual tests isolate each mechanic against the shipped store: the cascade leaves an artifact with no operative row, a discarded amendment leaves the row beneath it standing, a non-compromise revocation does not reach behind a signature, a countersignature whose window misses signed_at rescues nothing, and an artifact signed by a clean key is still downgraded by its channel. Two more build standalone variant stores from the shipped schema and rerun the agent's own binary against those JDBC URLs, so a hand-written manifest cannot pass. A final check confirms the shipped store still matches after the variant runs, which keeps the suite order independent.
