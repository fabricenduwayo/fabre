The build looks like a version bump but it is a constraint puzzle. The obvious moves each fail a different way: newest everything compiles and then dies at runtime because wire-format 3.x renamed the frame calls, accepting telemetry's declared transitive leaves wire-format below the security floor, and the oldest safe telemetry is missing a method the code calls. The agent has to reason across a security floor table, a runtime compatibility matrix, and one shared wire version, then override a transitive rather than trust any single pom. Reproducibility adds a second axis, since byte-identical rebuilds need Maven's output timestamp handling that most one-pass fixes skip. Everything must resolve offline from the bundled repository, so guessing versions that are not there fails immediately.

Pin the assignment the constraints force in the parent dependencyManagement: telemetry-client 1.4.1 for the batch API, wire-format 2.3.0 to clear the floor while staying on the 2.x runtime the clients need, and format-legacy 0.9.0 so the legacy path shares that wire. The override matters more than the pins, because telemetry declares wire 2.1.0 transitively and the build must resolve past it. Add project.build.outputTimestamp so consecutive clean builds are byte-identical, and drop the remote repository block from the cli pom since the policy forbids it. Then prove it end to end offline: mvn -B -o clean verify and a CLI run against the resolved jars.

Pytest drives the real Maven build offline and asserts on what it actually resolves, not on pom text. Policy tests parse dependency:list for the security floors, the single wire version, and the compatibility matrix, so no expected version is hardcoded anywhere; the constraint set forces the unique answer. The reproducibility test runs two clean builds and compares jar digests byte for byte. A runtime test resolves the cli classpath from the repository, executes the built CLI, and checks its output against the wire version the build bound, which catches bindings that compile but break at runtime. A last check confirms no pom declares remote repositories.
