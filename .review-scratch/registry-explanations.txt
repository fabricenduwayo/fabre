=== Difficulty ===
Agents must finish a Java reconcile CLI that joins live HTTP candidates with JDBC evidence while composing two stateful governance systems. Operative metrics require status filtering, supersession voiding, and temporal selection; waivers require an as-of event replay over version-scoped grants, revokes, and reciprocal replacement pairs, with expiry, malformed-pair rejection, non-revival, latest-grant selection, and partial suppression when multiple gates fail. Those results then alter multi-gate qualification and the H2-AUC tie-break. The shipped data includes overlapping grants, a valid replacement, a malformed transaction, an ordinary revoke, a future event, and an irrelevant waiver, so treating waivers as a simple allow-list produces the wrong promotion and manifest.

=== Solution ===
Implement `/app/reconcile-model-release` in Java: fetch registry candidates with HttpClient, derive operative validation metrics after voiding superseded runs, join calibration and version-scoped lineage over JDBC, and replay waiver_events through release_context.decision_at. Validate reciprocal replacement transactions, retain each waiver's latest valid state without predecessor revival, suppress only matching raw gate failures, and report only the selected grants that were actually used even when another failure still rejects the candidate. Then record H2-canonical conflicts, rank remaining qualifiers by H2 AUC and model id, validate the schema, and write the requested output for either the default or an alternate JDBC store.

=== Verification ===
Pytest recomputes the complete decision from the live API and H2 store, checking schema conformance, the five exact reason codes, latest-completed selection after supersession, conflicts, and the exact applied-waiver set. It reruns the real Java CLI against the shipped store and three verifier-built stores: one revokes a replacement and proves its predecessor does not revive, one combines a valid grant with a reciprocal but non-simultaneous replacement that must be ignored, and one gives omega simultaneous metric and lineage failures where only the metric waiver applies. These runs change the winner and reject hardcoded manifests, simple allow-lists, and shallow latest-row joins.
