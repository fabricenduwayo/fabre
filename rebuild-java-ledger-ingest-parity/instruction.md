The audit ingester that feeds our tamper-evident ledger is being decommissioned and nobody kept the source. All that is left is the built jar at `/opt/legacy/ledger-oracle.jar` and one sample export under `/app/samples/corpus-40` with the canonical rows and summary it produced, in `/app/samples/expected`. Write the replacement at `/app/ingest` so it agrees with the old one, because the audit trail is only trustworthy while both agree.

Branch files are pipe delimited as `seq|account|counterparty|amount|memo`, and branches are not careful about how they write them. The old ingester normalised whatever it was handed, rejected rows it could not attest, collapsed entries it considered the same claim, and wrote what survived into `canonical_ledger` plus a JSON summary of per-file and aggregate counts. Collapsing is where the money is: keep the wrong member of a group and a duplicated claim settles twice, and a branch that controls how it encodes a file controls which entries you treat as identical. Match the old ingester on the sample and on any other export the branches send, because the sample is one export and not the contract. The jar will not be on the box when your ingester runs, so work out what it does rather than calling it.

Build with `/app/ingest/build.sh` and run `bash /app/ingest/run.sh <corpus-dir> <jdbc-url> <summary-json>`. Rows are written from and to whichever store it was given, and the schema is at `/app/ledger-db/schema.sql`.
