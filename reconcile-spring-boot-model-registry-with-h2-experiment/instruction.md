The model-registry API and the H2 experiment store disagree about several churn-model candidates. Reconcile both into `/app/build/release-decision.json` using `/app/policy/promotion-policy.md` and `/app/policy/promotion-policy-amendments.md` (amendments override the body per section 1.4) plus `/app/schemas/release-decision.schema.json`. Do not modify either source.

Candidates and aliases come from `http://localhost:8080` (`bash /app/start-registry.sh` if it is down). Canonical validation metrics, version-keyed feature-hash lineage, and calibration status come from `jdbc:h2:file:/app/experiment-db/experiments` (user `sa`, empty password). The export needs the promoted model id (or null), every non-promoted candidate under `rejected` with exact policy reason codes, and every registry/H2 disagreement on an H2-canonical field under `conflicts` with both values. Missing experiment rows fail closed; lineage must match the candidate's exact version, not just any historical row for the model id.

Wire up `/app/reconcile-model-release` so its runnable jar applies the amended policy to live API and JDBC evidence, validates against the schema, and writes the manifest from that reconciliation — not a precomputed answer. Optional first argument: H2 JDBC URL; optional second: output path (defaults above). Pointing it at a different compatible database must yield the correct decision for that data.
